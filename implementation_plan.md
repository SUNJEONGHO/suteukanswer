# 수능특강 문제 풀이 웹 서비스 구현 계획

이 문서는 수능특강 문제 및 풀이 해설을 제공하는 웹 서비스 구축을 위한 기술 구현 계획입니다. Next.js, MongoDB (MCP), Tailwind CSS를 기반으로 구축됩니다.

## 데이터베이스 스키마 (MongoDB)

회원가입/로그인 기능이 없으므로, 핵심 도메인인 **'문제(Problem)'** 컬렉션 하나로 구성할 수 있습니다.

### `problems` 컬렉션
```json
{
  "_id": "ObjectId",
  "subject": "String",       // (선택) 과목명
  "chapter": "String",       // 단원명 (예: "1. 지수와 로그")
  "problemNumber": "Number", // 문제 번호 (예: 1, 2, 3...)
  "contentHtml": "String",   // 관리자가 업로드한 풀이 해설 HTML 코드 전체
  "createdAt": "Date",       // 생성 일자
  "updatedAt": "Date"        // 수정 일자
}
```

> [!TIP]
> `chapter`와 `problemNumber` 조합으로 고유 인덱스(Unique Index)를 설정하여 같은 단원에 중복된 문제가 실수로 등록되는 것을 방지할 계획입니다.

## 페이지 구성

웹 앱은 사용자용 인터페이스와 관리자용 인터페이스로 나뉩니다. App Router(`app/` 디렉토리)를 사용합니다.

### 1. 사용자 페이지 (Client Pages)
*   **메인 페이지 (`/`)**
    *   **문제 검색 기능**: 
        *   단원 선택 (기존에 등록된 단원 목록을 DB에서 가져와 드롭다운으로 표시)
        *   문제 번호 입력 (숫자 입력)
    *   '검색' 버튼.
*   **문제 풀이 조회 페이지 (`/problem/[id]` 또는 `/problem?chapter=...&number=...`)**
    *   검색된 문제의 단원명과 문제 번호를 상단에 표시.
    *   미리 저장된 `contentHtml` 코드를 화면에 그대로 렌더링 (`dangerouslySetInnerHTML` 활용).
    *   검색 결과가 없을 시 "해당 문제를 찾을 수 없습니다." 안내.

### 2. 관리자 페이지 (Admin Pages)
*   **관리자 메인 (`/admin`)**
    *   등록된 전체 문제 목록 조회 (단원별, 번호별 정렬).
    *   풀이 내용 수정 및 삭제 기능.
*   **새 풀이 등록 페이지 (`/admin/upload`)**
    *   아래 "풀이 업로드 페이지" 챕터에서 상세히 설명합니다.

## 풀이 업로드 페이지 상세

관리자가 HTML 풀이를 등록하는 `/admin/upload` 페이지의 동작 방식 및 UI 계획입니다.

1. **UI 구성 (Tailwind CSS 활용)**
   * **단원명 입력**: 입력창(input)으로 만들되, 기존에 입력했던 단원명들이 드롭다운으로 자동완성/제안되도록 하여 오타를 방지합니다.
   * **문제 번호 입력**: 숫자만 입력 가능한 입력창.
   * **HTML 에디터**: 
     * 왼쪽에는 넓은 텍스트 영역(`<textarea>`)을 두어 HTML 코드를 직접 붙여넣을 수 있게 합니다.
     * 오른쪽에는 **실시간 미리보기(Live Preview)** 영역을 제공하여, 작성 중인 HTML 태그가 실제로 화면에 어떻게 보일지 즉시 확인할 수 있도록 합니다.
2. **동작 로직**
   * '등록' 버튼 클릭 시 단원명, 문제번호가 채워져 있는지 유효성 검사.
   * Next.js API Route (`POST /api/problems`)를 호출하여 MongoDB에 저장 (MongoDB MCP를 통해 통신하거나 직접 MongoClient 사용).
   * 업로드 완료 후 성공 메시지를 띄우고 `/admin` 목록 페이지로 이동.

## Verification Plan

1. **환경 세팅**: `npx create-next-app@latest` 실행 (Tailwind CSS 포함).
2. **DB 연동**: MongoDB 연결 및 스키마 기반 데이터 통신 확인.
3. **업로드 페이지 테스트**: `/admin/upload`에서 실제로 임의의 HTML을 넣어 문제가 정상 등록되는지 테스트.
4. **조회 페이지 테스트**: 메인 화면에서 단원명/번호로 검색했을 때 등록한 HTML이 예쁘게 렌더링되는지 브라우저에서 직접 확인.
